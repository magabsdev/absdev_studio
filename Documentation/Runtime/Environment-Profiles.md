# Environment Profiles

## Purpose

Runtime profiles keep executable paths, database driver selection, and environment identity scoped to one project.

## How to use it

1. Open Runtime Centre.
2. Set the environment name and database driver.
3. Save the profile before running commands.

## Operational guidance

- The selected project is authoritative for project-aware commands.
- Review command output and diagnostics before proceeding.
- Use version control and backups before destructive operations.
