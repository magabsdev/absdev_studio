# Schema Compare

## Purpose

Compare two database schemas to identify missing or changed tables, columns, and indexes.

## How to use it

1. Choose source and target connections.
2. Run comparison.
3. Review generated migration guidance before applying it.

## Operational guidance

- The selected project is authoritative for project-aware commands.
- Review command output and diagnostics before proceeding.
- Use version control and backups before destructive operations.
