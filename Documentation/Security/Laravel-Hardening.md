# Laravel Hardening

## Purpose

Review Laravel production configuration, authentication, authorization, validation, logging, and dependency posture.

## How to use it

1. Disable debug in production.
2. Enforce authorization at boundaries.
3. Keep framework and packages supported.

## Operational guidance

- The selected project is authoritative for project-aware commands.
- Review command output and diagnostics before proceeding.
- Use version control and backups before destructive operations.
