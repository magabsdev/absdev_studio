# Database Problems

## Purpose

Diagnose wrong-database selection, failed connections, migrations, and schema inspection.

## How to use it

1. Verify the selected project.
2. Review environment values.
3. Test with a read-only query.

## Operational guidance

- The selected project is authoritative for project-aware commands.
- Review command output and diagnostics before proceeding.
- Use version control and backups before destructive operations.
