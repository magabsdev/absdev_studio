# Database Safety

## Purpose

Protect project data by checking the selected project, environment, connection, and backup status before changes.

## How to use it

1. Never assume the current database.
2. Back up important data.
3. Prefer Laravel migrations for repeatable schema changes.

## Operational guidance

- The selected project is authoritative for project-aware commands.
- Review command output and diagnostics before proceeding.
- Use version control and backups before destructive operations.
