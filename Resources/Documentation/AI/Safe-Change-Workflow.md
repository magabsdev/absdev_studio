# Safe AI Change Workflow

## Purpose

Use preview, review, tests, and version control before accepting AI-generated code changes.

## How to use it

1. Commit or checkpoint current work.
2. Review every proposed file change.
3. Run targeted and full test suites.

## Operational guidance

- The selected project is authoritative for project-aware commands.
- Review command output and diagnostics before proceeding.
- Use version control and backups before destructive operations.
