# Secrets and Environment

## Purpose

Keep credentials out of source control, documentation, AI prompts, and exported logs.

## How to use it

1. Use .env for local secrets.
2. Review Git status before commits.
3. Rotate exposed credentials immediately.

## Operational guidance

- The selected project is authoritative for project-aware commands.
- Review command output and diagnostics before proceeding.
- Use version control and backups before destructive operations.
