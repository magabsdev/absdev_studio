# Architecture Drift

## Purpose

Compare current project structure with recorded or expected architectural rules.

## How to use it

1. Define or review project standards.
2. Run drift analysis.
3. Classify findings before changing code.

## Operational guidance

- The selected project is authoritative for project-aware commands.
- Review command output and diagnostics before proceeding.
- Use version control and backups before destructive operations.
