# Request Flow

## Purpose

Trace an HTTP request from route and middleware through controller, service, model, and response.

## How to use it

1. Select a route.
2. Inspect middleware and controller action.
3. Follow discovered dependencies.

## Operational guidance

- The selected project is authoritative for project-aware commands.
- Review command output and diagnostics before proceeding.
- Use version control and backups before destructive operations.
