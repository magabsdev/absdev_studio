# Recovery and Backups

## Purpose

Recover from failed commands or changes using Git, backups, logs, and project history.

## How to use it

1. Stop destructive operations.
2. Preserve logs and error output.
3. Restore from the safest verified checkpoint.

## Operational guidance

- The selected project is authoritative for project-aware commands.
- Review command output and diagnostics before proceeding.
- Use version control and backups before destructive operations.
